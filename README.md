I build things close to the metal and ship them to production.  
Low-level systems, cross-platform desktop apps, compilers, network stacks, embedded firmware — anything where performance and correctness actually matter.

I don't chase trends. I use what fits the problem: C when the machine needs to be spoken to directly, C++ when the architecture demands it, Rust when I can afford the compile times and need the guarantees, TypeScript when the web is the delivery mechanism.

12+ years of debugging things that shouldn't be broken, optimizing things that shouldn't be slow, and maintaining things that nobody documented.
